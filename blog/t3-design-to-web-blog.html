Design to web

Responsive and mobile first design concepts

A responsive website is designed so that regardless of the viewing device or any variances in screen size it will still appear and function in optimal way. Mobile first design means that a site is designed as a mobile (or small screened) site first with the intention that it will then also appear optimised for larger screen sizes from there on up. The mobile first strategy is important because it not only aligns with the responsive design concept, but also because of the ongoing increase in mobile traffic. This approach provides users with the most consistent cross-platform experience.

Frameworks

Frameworks, in development terms, are generally a library or set of prewritten, rhobust code that provide a platform for developers to design sites with. An example of this is the Skeleton, a framework for mobile first design. The main benefit for using frameworks are the amount of time that a developer saves rather than if they are having to write the code for a particular site from scratch. However, the downside can sometimes be the lack of control or flexibility the developer loses over the design, like with Wordpress.

Wireframing

A wireframe is a visual design plan of a site or a site project before a developer begins the process of designing it. Wireframe provide important details such as:

  • Site contents
  • Site navigation (Both internal and external)
  • Page layout and design details
