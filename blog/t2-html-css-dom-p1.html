HTML, CSS and the DOM

During the Audacity tutorial, there was a very good analogy that describes the framework for a webpage where they compare it to building a house. I really liked this analogy and it completely makes sense, but, if I were having to create my own analogy it would be as follows…

Imagine that building a webpage is like building a set of shelves on the wall that can grow as you need more room. Theoretically, these shelves can have as many columns and rows as you want which can then be broken into separate compartments. Think of these shelves and compartments as the invisible framework of a website which is created using HTML.

Once your framework has been established you can then begin to populate the shelves and compartments with various objects such as books, magazines or even photo frames with pictures in them. Although you can still use HTML to achieve this, CSS also allows you to make changes that affect the appearance of the various shelves, compartments and objects that are placed upon these shelves.

Let's say I want to place a photo or 'image' on the top shelf. CSS not only allows me to position it exactly where on the shelf I want it to sit but I can also decide what kind of frame or border I want to surround it. On the next shelf down I decide I want to place two photos and a book but I want them to look evenly spaced at 100 pixels apart. I also realise that one of the photos is too big to fit and so it will need to be resized to fit. To top it off, I want this part of the shelf to be painted a different colour. Once again, this is all achieved using CSS.

What is meant by boxifying design?

This is a reference to how a page on a website is put together. Everything on the page (objects, elements etc) is seen as a square or a box. These squares or boxes are then placed into the sections of the page which is also technically a box and these sections make up the page which is one big box.

Boxified design could also be applied or demonstrated in the shelf analogy that was used in the first part of this exercise.

What is the box model?

The ‘Box Model’ is a term used when referring to the way a website design and layout is created using CSS and is considered to be the foundation of layout on the web. 

Essentially, each element on our page is a perceived as a a rectangular box and it contains our content. Surrounding our content within our box are a number of CSS layers (padding, border and margin) which can all be manipulated to affect how our content appears or behaves.
